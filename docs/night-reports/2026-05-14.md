## What happened today (before night run)

The day was dominated by the recency-policy hunt:

- Magazine showed a "Top 5 Horror Movies" trend traced to a 21-day-old
  Epidemic Sound blog. That kicked off a deeper article-date audit.
- Found 170 trends backed by source articles older than 14 days, some
  up to 2831 days (8-year-old Allure piece on tragus piercings, 5-year
  Cosmopolitan listicle that produced the Owala FreeSip trend, etc.).
- Built three defence layers: SQL filter on old-year URLs in magazine
  + dashboard queries; HTTP-fetch verifier that parses meta tags
  (Open Graph, JSON-LD, Schema.org microdata, HTML5 `<time>`); daily
  cron step that runs the verifier automatically.
- Raised the recency cap from 14 → 30 days because trend-roundup blogs
  often describe trends that are still very much going. Reactivated
  61 trends that fell in the 14-30 day window.
- Reddit JSON scraper added (Firecrawl blocks reddit.com) — turned out
  Reddit also 403's all Vercel cloud IPs. Both JSON and RSS endpoints.
  Replaced with 8 Perplexity-based subculture queries.
- Dashboard search broke because default limit was 100 and the trend
  the user searched for had popularity 7. Bumped default to 1000.
- Built a live scrape-progress panel (bottom-right of dashboard) with
  DB-backed job tracking that survives page refresh + tab close.

### Phase 5: Cleanup + bugfixes — DONE

**Fix 1 — moments_query Bad Request:** 5 sources (`Perplexity — Award
shows`, `— Music drops`, `— TV finales`, `— Product launches`,
`— Celebrity moments`) plus `Spotted in the Wild` were failing every
scrape with "Bad Request". Root cause: their `source_type` is
`perplexity_moment_query` (for /api/moments/fetch, a separate
forward-looking calendar pipeline), but /api/culture/scrape was loading
ALL active sources and trying to Firecrawl their pseudo URLs
(`internal://perplexity/moments-*`). Fixed by filtering both
`perplexity_moment_query` and `manual` out of the default scrape sweep
(commit `6036f20`).

**Fix 2 — article-date extractor coverage:** checked the hosts that
return HTTP 200 + source=none. Mostly Pinterest, genz.ai listing
pages, tools.explodingtopics.com — none of these are individual
articles with publication dates. The current "inconclusive" handling
is correct: leave those trends alone, don't archive them.
